We are now enterprise-safe and certification-clean (70/70 tests, 7 gates). The next sprint must focus on increasing predictive power and AI usefulness without compromising safety. Implement a two-track roadmap: (A) predictive evaluation + calibration, and (B) insight-only AI copiloting that drafts actions but never executes without approvals.

First, add an “Offline Evaluation & Calibration” subsystem. Create an evaluation harness that runs on historical data and outputs a versioned report (markdown + JSON artifact) with forecast metrics (WAPE/sMAPE, bias, intermittent demand handling), allocation metrics (fill rate, backorder reduction, cost), and procurement timing metrics (estimated vs measured savings, with explicit separation). Include confidence calibration: reliability curves and bucketed accuracy so that confidenceThreshold has a measurable meaning. Add drift detection that monitors rolling error and data distribution shifts; when drift is detected, automatically downgrade autonomy to “suggest,” log a structured_event_log entry, and surface an in-app warning.

Second, implement an “Insight-Only AI Copilot.” The AI must be constrained to read-only and draft-only behaviors. It can answer natural language questions by querying our database and returning summaries with evidence (entity IDs, timestamps, and counts), and it can generate action drafts (purchase order draft, RFQ draft, safety stock adjustment proposal, inventory rebalance plan) but must never mark actions as completed unless a real system action occurred. Drafts should be stored as ai_actions with status=pending or awaiting_approval, include estimatedImpact with confidence, and link to the triggering context (regime snapshot, forecast run, low-stock event). Execution must remain gated behind safe mode, spend limits, guardrails, and approval workflows exactly as in the automation engine.

Third, improve decision intelligence with a policy layer that translates regime + forecast uncertainty + operational constraints into recommended parameters (quantity, timing, supplier choice). Add what-if simulation endpoints that compute projected service level, stockout risk, cash impact, and lead time risk under different decision options. Every override must be logged in behavioral tables with factual context to support future learning without nudging.

Fourth, strengthen the data foundation: implement supplier/material lead-time distributions, MOQ/pack/capacity constraints as first-class inputs, and a data quality scoring system (missingness, staleness, outliers). Automation must be blocked or downgraded when data quality falls below thresholds, with structured logging and user-visible reasons.

Deliverables required: (1) evaluation harness command and artifacts (JSON + markdown) that run deterministically in CI, (2) AI copilot read-only/draft-only mode with evidence-first outputs, (3) drift detection + autonomy downgrade wiring, (4) what-if simulation endpoints, and (5) tests proving no execution can occur through AI without approvals and no phantom “completed” states can be produced. Ensure all new functionality preserves the existing 7-gate certification and hardening tests without regression.
