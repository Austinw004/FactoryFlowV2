SYSTEM DIRECTIVE: IMPLEMENT END-TO-END PROCUREMENT EXECUTION + BILLING + PAYMENT SYSTEM

GOAL:
Allow users to approve a recommended purchase and immediately execute a real supplier transaction securely, with full auditability and enterprise-grade billing controls.

This must:
- Be secure (no raw card storage)
- Be auditable (every action logged)
- Be trust-guarded (no unsafe automation)
- Support both direct payment and PO-based workflows

========================================================
SECTION 1 — USER FLOW (CRITICAL)
========================================================

FROM UI:
Pending Actions Tab → Recommended Purchase → "Approve & Execute"

FLOW:

1. User clicks "Approve & Execute Purchase"
2. System:
   - validates trustScore >= 0.6
   - checks dataQualityScore >= threshold
   - ensures supplier + pricing exists

3. If valid:
   → create PurchaseIntent
   → trigger payment OR PO flow
   → log audit trail

========================================================
SECTION 4 — PAYMENT EXECUTION (STRIPE)
========================================================

FUNCTION: executePurchase(purchaseIntentId)

STEPS:

1. Fetch purchaseIntent
2. Fetch billingProfile
3. Compute totalAmount

4. FRAUD CHECK:
   - run computeFraudScore()
   - if >0.9 → BLOCK
   - if >0.7 → require approval

5. CREATE Stripe PaymentIntent:
   - amount = totalAmount
   - customer = stripeCustomerId
   - payment_method = defaultPaymentMethodId
   - confirm = true

6. HANDLE RESULT:
   - success → status = completed
   - failure → status = failed

7. STORE transaction record

========================================================
SECTION 5 — PURCHASE APPROVAL LOGIC
========================================================

FUNCTION: approveAndExecuteRecommendation(recommendationId)

VALIDATIONS:

- trustScore >= 0.6
- NOT automationBlocked
- supplier exists
- unitCost exists (NOT NaN)
- quantity > 0

CREATE purchaseIntent:

- status = approved
- link to recommendation

CALL:
→ executePurchase()

SECTION 8 — FRAUD + TRUST GUARDS
========================================================

BEFORE EXECUTION:

BLOCK IF:

- trustScore < 0.6
- unitCost is NaN
- quantity unrealistic (>30× lead-time demand)
- fraudScore > 0.9

REQUIRE APPROVAL IF:

- fraudScore > 0.7
- trustScore between 0.4–0.6
