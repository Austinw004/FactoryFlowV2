## 3. CROSS-FEATURE INTEGRATION ANALYSIS

### How Scenario Planning + Geopolitical Risk Work Together

**Integration Effectiveness: 9/10**

**Synergies**:
1. **Geopolitical Event → Scenario Trigger**: 
   - High-risk geopolitical assessment (85/100 score) triggers scenario planning simulation
   - Example: "Trade War: High Severity" → "Run Scenario: Commodity Price +40%, Demand -20%"

2. **FDR Context Shared**:
   - Both features use the same FDR calculation (Ma*Va / Mr*Vr)
   - Geopolitical events modify regional FDR values
   - Scenario planning uses adjusted FDR for regime determination

3. **Confidence Correlation**:
   - Geopolitical confidence (70-80%) feeds into scenario confidence (65-75%)
   - If geopolitical assessment has 90% confidence → scenario uses tighter parameter bounds

**Real-World Example**:
```
1. Geopolitical: "US-China Trade War, High Severity, 85% confidence"
   - Region: Asia Pacific
   - Commodities Affected: Semiconductors, Rare Earths
   - Risk Score: 85/100

2. Triggers Scenario Planning:
   - Event Type: "Supply Chain Disruption"
   - FDR Delta: +0.2 (financial markets react faster than production)
   - Commodity Price Change: +45% (semiconductors)
   - Demand Change: -15% (consumer electronics slowdown)
   - Duration: 18 months

3. Combined Result:
   - New Regime: "Asset-Led Growth" (FDR 1.35)
   - Financial Impact: -$4.2M revenue
   - Recommended Action: Diversify suppliers to Vietnam/India, hedge semiconductor prices
```

**Why This Integration is Powerful**:
- ✅ **Causal Chain**: Geopolitical events cause economic regime shifts → better than treating them independently
- ✅ **Automated Workflow**: High geopolitical risk auto-generates relevant scenarios
- ✅ **Unified Dashboard**: Executives see both "what might happen" and "what it means for our business"

4. **Comprehensiveness** (9/10)

   - ✅ Geopolitical Risk: 4 event types, 6 regions, 3 severity levels
   - ✅ Together cover both macro-economic and geopolitical risk surfaces

#### Weaknesses (Where Improvement Needed)

1. **Backtesting Validation** (6/10)
   - ⚠️ Confidence scores (65-75%, 70-80%) are theoretical - need historical validation
   - ⚠️ Missing: "In 2018-2020, did 70% confidence predictions match 70% accuracy?"
   - **Recommendation**: Run predictions against 2015-2023 historical data, calculate actual accuracy

2. **Calibration** (6.5/10)
   - ⚠️ Average financial impact needs validation against actual business outcomes
   - ⚠️ Risk scores (55-65/100) need benchmarking to known events
   - **Recommendation**: Calibrate to known outcomes (e.g., 2018 trade war = 75 risk score retrospectively)

3. **Granularity** (7/10)
   - ⚠️ Risk impact buckets (<$1M, $1M-$3M, >$3M) too broad for large enterprises
   - ⚠️ Regional analysis needs sub-regional detail (China vs. India vs. Southeast Asia)
   - **Recommendation**: Add $5M, $10M, $25M thresholds; split Asia Pacific into 3 sub-regions

4. **Time Series** (6/10)
   - ⚠️ Bulk testing is snapshot-based - doesn't test regime transitions over time
   - ⚠️ Missing: "If FDR rises from 1.0 → 1.3 over 6 months, what's the procurement strategy?"
   - **Recommendation**: Add time-series scenario modeling (monthly FDR path generation)
